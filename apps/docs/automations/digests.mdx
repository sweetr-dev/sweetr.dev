Digests can be sent monthly, weekly, or daily. Digests are team scoped, and must be enabled individually for each team.

## Metrics Digest

<Frame>
  <img src="/images/digest-metrics.png" />
</Frame>

The metrics digest includes a summary of metrics found in [Pull Request Flow](/metrics-and-insights/pr-flow/intro) and [Code Review Efficiency](/metrics-and-insights/code-review-efficiency/intro).

- Average Pull Request Size
- Average Cycle Time
  - Time to First Review
  - Time to Approve
  - Time to Merge

Enabling this digest helps teams understand their flow and areas for improvement. It removes the hurdle of having to manually check the metrics in the Sweetr dashboard.

## Work in Progress Digest

<Frame>
  <img src="/images/digest-wip.png" />
</Frame>

The work in progress digest includes a summary of all the team's open Pull Requests found in [Work in Progress](/metrics-and-insights/wip).

Enabling this digest helps team alignment, by bringing visibility to any PRs that may need attention.
